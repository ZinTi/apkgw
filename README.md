# apkgw

apkgw 是一款运行于 Android 手机的 **代理网关（Proxy Gateway）** 软件。它能让连接到手机 Wi-Fi 热点的其他设备（如笔记本电脑、平板）通过手机当前已开启的代理通道访问互联网，而无需在那些设备上安装任何代理客户端软件。

> **请注意**：apkgw 本身 **不是代理服务提供方** ，也不提供任何加密、隧道或翻墙功能。它只是一个标准的 HTTP/HTTPS 代理转发器，将客户端的代理请求原样转发至目标服务器。若手机系统已配置网络代理（例如通过第三方代理软件实现），则所有转发流量将自动经由该代理通道出口，从而实现网络访问的灵活路由。

---

## 功能特性

- **标准代理支持**：
  - 支持 HTTP `CONNECT` 隧道（用于 HTTPS、gRPC over TLS 等加密流量）。
  - 支持标准 HTTP 代理（`GET http://host/path` 等请求），兼容普通 HTTP 访问。
- **高性能并发**：可同时处理多个客户端连接（默认最大 200 个），采用异步非阻塞 I/O，主界面不卡顿。
- **Android 前台服务**：启动后以后台服务运行，并显示持续通知，防止 Android 8.0+ 系统杀死进程，保证长时间稳定运行。
- **保活机制**：持有 `PARTIAL_WAKE_LOCK` 唤醒锁，避免息屏后 CPU 休眠导致断流。
- **简洁易用的界面**：
  - 显示当前热点 IP 地址和监听端口。
  - 一键开关服务，状态指示灯实时反馈。
  - 实时滚动日志窗口，便于调试和监控。
  - 支持清空日志、复制日志到剪贴板。
- **端口可配置**：允许用户自定义监听端口（默认 8888）。

## 使用方式

### 前提条件
- 一部 Android 手机（已开启热点功能）。
- 希望上网的设备（如笔记本电脑）需要能连接该热点。

### 步骤

1. **在手机上安装并运行 apkgw**。
2. **开启热点**（系统设置中打开“便携式热点”），确保电脑已连接到该热点。
3. **在 apkgw 应用中**：
   - 查看界面顶部显示的 **热点 IP**（例如 `192.168.43.1`）。
   - 点击开关启动服务（默认端口 `8888`）。
4. **在电脑上配置代理**：
   - 将电脑的系统代理或应用程序代理设置为该 IP 和端口。
   - 以 Chromium 浏览器为例，在终端中执行：
     ```bash
     chromium --proxy-server="http://192.168.43.1:8888"
     ```
   - 对于命令行工具（如 `curl`）：
     ```bash
     export http_proxy="http://192.168.43.1:8888"
     export https_proxy="http://192.168.43.1:8888"
     curl https://cip.cc
     ```
   - 其他应用（如 Firefox、Edge、系统网络设置）同理，将 HTTP/HTTPS 代理地址填入该 IP 和端口。

5. **开始上网**：电脑上所有经过代理的流量都会通过手机转发。若手机已配置网络代理，则流量会自动经由该代理出口。

## 编译与安装

### 获取源码
```bash
git clone https://github.com/zinti/apkgw.git
cd apkgw
```

### 修改配置（可选）
- 默认包名为 `com.github.zinti.apkgw`，如需自定义，请按 Android 规范修改。
- 默认监听端口 `8888`，可在应用内动态修改。

### 构建 APK
确保已安装 Flutter SDK 和 Android 开发环境，执行：
```bash
flutter clean
flutter pub get
flutter build apk --release
```
生成的 APK 位于 `build/app/outputs/flutter-apk/app-release.apk`。

### 安装
将 APK 传输到手机并安装，或使用 `adb install`。

## 注意事项

- **明文流量**：`AndroidManifest.xml` 中已开启 `android:usesCleartextTraffic="true"`，允许接收 HTTP 明文请求（因为代理 CONNECT 请求本身是明文的）。
- **Android 版本兼容**：支持 Android 5.0（API 21）及以上，前台服务适配 Android 14（API 34）的 `dataSync` 类型。
- **唤醒锁**：应用会持有唤醒锁以维持网络连接，会略微增加耗电，建议仅在需要时开启服务。
- **安全性**：该代理不加密内容，请确保热点环境可信，避免敏感信息泄露。

## 工作原理

apkgw 在手机上启动一个 TCP 服务器，监听指定端口。当收到客户端发来的代理请求时：

1. **如果是 `CONNECT` 方法**（用于 HTTPS），则与目标服务器建立 TCP 连接，并在客户端和目标之间建立双向数据转发隧道。
2. **如果是 `GET http://...` 等标准 HTTP 代理请求**，则解析出目标主机和端口，连接目标服务器，将请求行重写为相对路径后转发，并继续双向转发后续数据。

所有 Socket 连接均通过系统的网络栈发出，因此如果手机已配置网络代理（如通过系统设置或第三方软件），则系统会自动将这些连接路由至代理出口，实现设备无感网络访问。
